# GPA Calculator 功能升级路线图

这个项目已经完成了 GPA 计算、学生档案、目标院校、本地规则分析和 Vercel AI 代理。下一步不建议堆很多零散按钮，而是把它升级成一个长期可用的“升学规划驾驶舱”。

## 产品定位

从“算 GPA”升级为“管理大学期间的升学竞争力”：

- 成绩是底盘：GPA、排名、学分结构、核心课表现。
- 经历是资产：科研、竞赛、项目、英语、奖项、材料。
- 目标是约束：本校保研、外校推免、考研、出国、就业，每条路要求不同。
- 行动是核心：系统不只判断强弱，还要告诉用户下一周、下个月该补什么。

## 优先级 1：先把现有工具变得更好用

这些功能成本低，但会明显提升使用体验。

### 1. GPA 趋势图

新增可视化：

- 每学期 GPA 折线图
- 累计 GPA 折线图
- 总学分增长趋势
- 低绩点课程提醒

价值：用户一眼能看出自己是在变好还是变差。

### 2. CSV 导入

目前已经支持导出 CSV，下一步应该支持导入。

字段建议：

- 课程名称
- 学期
- 学分
- 分数

价值：换电脑、重装浏览器、批量录入成绩时更方便。

### 3. JSON 备份和恢复

CSV 只适合课程，JSON 可以备份完整数据：

- 课程
- 学生档案
- 目标院校
- AI 接口配置
- 本地规则分析版本

价值：避免 localStorage 换浏览器丢失。

### 4. 成绩影响分析

新增“如果这门课考多少分，会把总 GPA 拉到多少”的模拟器。

场景：

- 一门 5 学分课考 90 / 93 / 96 分分别影响多少？
- 未来某学期要达到 4.3，需要每门课大概多少？

价值：比单纯目标 GPA 更贴近真实学习决策。

## 优先级 2：升学规划核心能力

这些功能会把项目从计算器变成规划工具。

### 1. 目标院校对照表

每个目标院校显示一张对照卡：

- 当前 GPA vs 目标最低 GPA
- 当前排名 vs 目标最低排名
- 当前英语 vs 英语要求
- 当前科研/竞赛 vs 偏好
- 材料准备状态

输出：

- 已满足
- 接近
- 风险
- 信息不足

价值：用户不是看抽象概率，而是知道自己差在哪里。

### 2. 多路径评分

同一个人对不同方向的竞争力不同，应分开评分：

- 本校保研评分
- 外校推免评分
- 考研评分
- 出国评分
- 就业评分

评分逻辑示例：

- 本校保研更看排名、绩点、学院政策。
- 外校推免更看排名、英语、科研、材料。
- 考研更看初试科目、复习进度、目标分差。
- 出国更看 GPA、语言、科研/实习、预算和时间线。
- 就业更看项目、实习、技能栈、作品集。

价值：避免用一个总分粗暴判断所有未来。

### 3. 短板雷达

把短板分成 6 个维度：

- GPA
- 排名
- 英语
- 科研
- 竞赛/项目
- 材料准备

每个维度给：

- 当前等级
- 风险原因
- 下一步动作

价值：让用户知道“现在最该补什么”，而不是焦虑。

### 4. 时间线与截止日期

新增“升学时间线”：

- 大一：稳 GPA、适应课程、尝试项目
- 大二：补英语、找科研、参加竞赛
- 大三：确定目标、参加夏令营、准备材料
- 大四：推免、考研、申请、就业

用户也可以自己添加：

- 夏令营报名截止
- 英语考试日期
- 竞赛报名日期
- 材料提交日期

价值：规划从“分析结果”变成“行动日历”。

## 优先级 3：智能分析增强

这些功能适合在 OpenAI API key 配好之后做。

### 1. 严师型周报

每周自动或手动生成：

- 本周 GPA/课程风险
- 本周材料推进
- 本周目标变化
- 下周 3 个硬任务

语气保持现在设定：直给、严格、行动导向。

### 2. 目标院校材料审阅

用户输入或粘贴：

- 简历
- 个人陈述
- 科研经历描述
- 项目经历描述

AI 输出：

- 哪些地方空
- 哪些地方太虚
- 哪些经历应该量化
- 哪些句子不适合申请材料

价值：这会非常实用，比单纯“聊天分析”强。

### 3. 问答式升学导师

新增一个轻量聊天区，但不做泛泛聊天。

它只回答这些范围：

- 如何补 GPA
- 如何选目标院校
- 如何准备材料
- 如何拆解科研/竞赛
- 如何安排未来 1-4 周行动

价值：把 AI 从报告生成器变成可追问的导师。

## 优先级 4：数据模型升级

当功能变多后，建议把数据结构拆得更清楚。

### 1. 课程增加分类

新增字段：

- 课程类别：必修、限选、任选、通识、体育、实践
- 是否计入平均学分绩点
- 是否核心课
- 备注

价值：有些学校实践环节或特殊课程不计入 GPA，这个会更准确。

### 2. 目标院校增加状态

新增字段：

- 冲刺 / 匹配 / 保底
- 已查政策 / 未查政策
- 已准备材料 / 未准备材料
- 联系导师状态
- 报名截止日期

价值：目标院校从“列表”变成“申请项目管理”。

### 3. 经历库

新增独立模块：

- 科研经历
- 竞赛经历
- 项目经历
- 实习经历
- 奖项证书

每条经历包含：

- 名称
- 时间
- 角色
- 产出
- 证明材料
- 可用于哪些目标

价值：以后写简历、AI 分析、材料审阅都能复用。

## 推荐的下一版

最建议先做 `v2.1`：

1. GPA 趋势图
2. CSV 导入
3. JSON 完整备份/恢复
4. 成绩影响模拟器
5. 目标院校对照表

原因：这些不依赖 OpenAI key，马上能提升网站可用性，也不会把项目复杂度一下推爆。
